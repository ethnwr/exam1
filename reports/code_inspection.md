## Нарушение 1. Неправильное именование переменных

| | |
|---|---|
| Файл | `src/Module2_Cart/cart.py` |
| Строка | в методе `add()` |
| Описание | Переменная `pid` вместо `product_id` — сокращение затрудняет чтение |
| Почему плохо | Непонятно, что такое `pid` без контекста |
| Исправлено | `pid` заменено на `product_id` во всём файле |
| Коммит | `refactor: переименованы сокращённые переменные в cart.py` |

## Нарушение 2. Отсутствие комментариев к методам

| | |
|---|---|
| Файл | `src/Module3_Order/order.py` |
| Строка | функции `validate_customer()` и `create_order()` |
| Описание | Нет docstring, описывающих что делает функция и что возвращает |

| Коммит | `docs: добавлены docstring в order.py` |

## Нарушение 4. Дублирование кода

| | |
|---|---|
| Файл | `src/Module4_Admin/admin.py` и `src/Module1_Catalog/catalog.py` |
| Строка | функции загрузки данных |
| Описание | В обоих модулях есть своя функция `load_products()` с одинаковым кодом |
| Почему плохо | При изменении формата JSON нужно править в двух местах |
| Исправлено | `admin.py` теперь импортирует `load_products` из `catalog.py` |
| Коммит | `refactor: устранено дублирование загрузки каталога` |
